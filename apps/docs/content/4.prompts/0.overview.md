## 什么是提示词？

提示词是可复用的消息模板，可供 AI 助手使用。它们可以包含动态参数，并返回预先格式化的消息。

::prompt
---
description: 为新的 MCP 提示词搭建脚手架
icon: i-lucide-message-square
actions:
  - copy
  - cursor
  - windsurf
---

使用 @nuxtjs/mcp-toolkit 在我的 Nuxt 应用中创建一个新的 MCP 提示词。

- 在 server/mcp/prompts/ 中创建一个文件（例如 server/mcp/prompts/review-code.ts）
- 使用 defineMcpPrompt（自动导入）并提供描述和处理器
- 处理器可以返回一个字符串（会自动包装为用户消息）或一个包含 messages 数组的完整 GetPromptResult
- 在 inputSchema 中使用 `import { z } from 'zod'` 定义参数（提示词参数必须是字符串）
- 使用 role 选项（'user' 或 'assistant'）来控制字符串返回时的默认角色
- 使用 completable() 为参数提供自动补全建议
- 返回多个消息以创建对话流程
- 名称和标题会根据文件名自动生成
- 当在聊天中输入 / 时，提示词会出现在 Cursor 和 VS Code 中

文档：https://mcp-toolkit.nuxt.dev/prompts/overview

::

## 为什么使用提示词？

与临时性的指令相比，MCP 提示词提供了几个优势：

::card-group
  :::card
  ---
  icon: i-lucide-repeat
  title: 可复用性
  ---
  定义一次，到处使用。在团队中共享提示词，以获得一致的 AI 交互。
  :::

  :::card
  ---
  icon: i-lucide-check-square
  title: 标准化
  ---
  确保特定任务（如代码审查或文档编写）具有一致的格式和上下文。
  :::

  :::card
  ---
  icon: i-lucide-settings-2
  title: 自定义
  ---
  使用参数将提示词适配到不同上下文，同时保持结构不变。
  :::

  :::card
  ---
  icon: i-lucide-plug
  title: IDE 集成
  ---
  提示词会出现在 Cursor、VS Code 和 Visual Studio 中，便于在开发过程中快速访问。
  :::
::

## IDE 集成

当你的 MCP 服务器已连接时，MCP 提示词可以与现代开发环境无缝集成，提示词会直接在你的 IDE 中可用。

### 在 Cursor / VS Code 中使用提示词

1. **输入 `/`**：在聊天中输入 `/` 以查看所有可用的 MCP 提示词
2. **选择一个提示词**：从列表中选择（例如，`local-mcp/setup-mcp-server`）
3. **填写参数**：对于提示词模板，会弹出一个对话框来填写所需参数
