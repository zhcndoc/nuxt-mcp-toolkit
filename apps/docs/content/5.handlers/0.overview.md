## 什么是处理程序？

处理程序允许你在单个 Nuxt 应用中创建**多个 MCP 端点**。每个处理程序都有自己的路由、名称、版本，并且可以包含自己的一组工具、资源和提示词。

使用 @nuxtjs/mcp-toolkit 创建一个带有自己的端点和工具的自定义 MCP 处理程序。

- 在 server/mcp/ 根目录下创建一个 .ts 文件（例如 server/mcp/admin.ts）——子目录中的文件，如 tools/、resources/、prompts/，是定义，而不是处理程序
- 使用 defineMcpHandler（自动导入），并提供 name、route、tools、resources 和 prompts
- 显式设置 route（例如 route: '/mcp/admin'）
- 使用 defineMcpTool 内联定义工具，或导入共享工具
- 通过创建 server/mcp/index.ts 并使用 defineMcpHandler 来覆盖默认处理程序
- 为处理程序特定的身份验证或日志记录添加中间件
- 为每个处理程序设置 version 和 browserRedirect

文档：https://mcp-toolkit.nuxt.dev/handlers/overview

::

这在你想要以下情况时很有用：
- 将不同的 MCP 功能拆分到不同的端点中
- 创建带版本的 MCP API
- 按领域或功能组织工具/资源
