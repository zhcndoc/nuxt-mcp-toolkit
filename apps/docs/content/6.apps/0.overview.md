## 什么是 MCP 应用？

MCP 应用是由 MCP 工具返回的 **交互式 HTML 小部件**，并由兼容的主机以内联方式渲染。你的工具不会流式返回文本，而是提供一个用户可以阅读、滚动、筛选和点击的小型 UI——通过一个带类型的消息桥接与服务器重新连接。

它们遵循 [MCP UI 提案（SEP-1865）](https://modelcontextprotocol.io)：工具返回一个由 `ui://` 引用的 `text/html;profile=mcp-app` 资源，主机将其加载到一个沙箱化的 `iframe` 中，然后 iframe 通过 `postMessage` 进行回传通信。

`@nuxtjs/mcp-toolkit` 让这种编写体验就像在写一个普通的 Nuxt 页面：

- 每个应用在 `app/mcp/` 中对应一个 **Vue SFC**。
- `<script setup>` 中的 `defineMcpApp` 宏声明工具、schema 和服务器处理程序。
- `useMcpApp()` 可组合函数为你提供响应式数据、主机上下文，以及到主机的类型化桥接。
- 该工具包会在构建时将 SFC 打包为一个自包含的 HTML 文件，并通过你的 MCP 端点提供它。

使用 @nuxtjs/mcp-toolkit 在我的 Nuxt 应用中创建一个新的 MCP 应用。

- 在 app/mcp/ 中创建一个 Vue SFC（例如 app/mcp/color-picker.vue）
- 在 <script setup lang="ts"> 中使用 defineMcpApp({ description, inputSchema, handler })
- inputSchema 使用 Zod；handler 在服务端运行并返回 { structuredContent }
- handler 可以通过 $fetch 调用任何 Nuxt API 路由——将繁重的数据处理保留在服务端
- 在模板中，调用 useMcpApp<TPayload>() 以获取 { data, loading, hostContext, sendPrompt, callTool, openLink }
- 首次渲染时，data 会从 structuredContent 中进行 hydrate——无需额外请求
- 使用 sendPrompt(text) 将后续内容推送到聊天中，从而触发另一个工具/应用
- 使用 callTool(name, params) 重新调用一个 MCP 工具并在原位刷新数据
- 使用 openLink(url) 请求主机在 iframe 外打开一个 URL
- 如果你要加载图片或调用外部 API，请使用 csp: { resourceDomains, connectDomains } 添加 CSP 白名单
- 要将应用路由到专用处理程序，可以将 SFC 放在 app/mcp/<handler>/ 下，或者在 defineMcpApp 上设置 attachTo: '<handler>'（字符串字面量）。group 和 tags 也接受字面量。
- 让布局保持流动式（不要固定高度）；主机通常会以可变宽度以内联方式渲染 iframe

文档：https://mcp-toolkit.nuxt.dev/apps/overview
